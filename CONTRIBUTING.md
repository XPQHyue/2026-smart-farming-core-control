# Contributing

感谢参与本项目。本文件简要说明如何构建、测试与提交贡献（PR）。

## 构建与编译

- 本工程使用 Keil uVision 5（ARMCC 或 ARM Compiler v6）进行构建。
- 打开工程：`USER/WHEELTEC.uvprojx`。
- 在 Keil 中选择 `Project -> Rebuild All`（或使用团队的自动化脚本）。

注意：仓库中保留了 `OBJ/`、`build/` 等编译产物在原工程中；当前公开仓库已移除大型二进制产物，提交前请确认不包含编译产物。

## 提交与分支策略

- 使用 Fork + Branch 流程：
  1. Fork 本仓库到你的账号
  2. 新建分支，命名格式 `feat/<短描述>` 或 `fix/<短描述>`
  3. 提交改动并推送到你的 fork
  4. 提交 Pull Request 到本仓库的 `main` 分支

- PR 说明请包含：目的、改动要点、如何验证（如果适用）和相关 issue（如有）。

## 代码风格

- C 代码：使用与本项目一致的缩进风格（4 空格），避免行尾多余空格，保持函数注释简洁。
- 不要提交编译生成的二进制文件或大型依赖（将其加入 `.gitignore`）。

## 问题报告

- 请在 Issues 中创建问题，附上重现步骤、日志及必要的环境信息（编译器版本、板卡型号、连接口等）。
